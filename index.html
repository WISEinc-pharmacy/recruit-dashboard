マニュアル・仕様

概要

外来プロジェクト（越谷・清瀬新店）の採用候補者を一元管理するダッシュボードです。結〈yui〉がrecruit@メールを5分ごとにポーリングし、新規候補者を自動登録します。Slackボタン操作もリアルタイム連動します。

データの流れ

  • 自動登録: recruit@にメール着信 → 結〈yui〉が5分以内にSlack投稿＋Firestoreに自動書き込み → ダッシュボードにリアルタイム反映
  • ステータス連動: Slackで「城北」「東葛」「全エリア」「不採用」ボタン押下 → Firestoreのエリア・ステータスが自動更新
  • 手動更新: ダッシュボード上で候補者をタップ → サイドパネルからステータス・エリア・メモ等を編集・保存

画面の見方

要素説明
上部カード（全件/面接確定/調整中/選考中/終了/返信要）タップでその区分にフィルタ。数字は該当件数
検索バー名前・JS ID・エリア・担当者名などフリーテキスト検索
エリアフィルタ城北/東葛/越谷/清瀬/足立で絞り込み
ソート更新日順（新/古）・年齢順・ステータス順
候補者行タップ右からサイドパネルが開き、詳細表示・編集が可能

サイドパネル（候補者詳細）

  • 上部: JS ID・名前・年齢/性別・居住地・担当者・経験・希望年収・入職時期（読み取り専用）
  • ステータス: 紹介受領 → 書類選考中 → 面接調整中 → 面接確定 → 面接済み → 入社確定。またはいつでも不採用/辞退/見送り/保留に変更可
  • エリア: 複数選択可（チェックボックス式）。城北/東葛/越谷/清瀬/足立
  • 外来PJ打診: 越谷・清瀬新店の外来プロジェクトを案内済みかどうか（エリアタグとは独立したフラグ）
  • 対応者: 社内で誰が対応中か記録
  • 返信要: 「要」= こちらからアクション必要、「待ち」= 先方返信待ち、「不要」= 対応不要
  • 備考メモ: 自由記述。面接日程・条件交渉内容・判断メモなどを記録
  • 編集後は保存ボタンで確定。保存前に閉じようとすると確認ダイアログが表示されます

ステータスの定義

ステータス意味
紹介受領エージェントから紹介を受けた初期状態
書類選考中履歴書・職務経歴書を確認中
面接調整中面接日程を調整中
面接確定面接日時が確定
面接済み面接実施済み・結果判断中
入社確定内定承諾・入社日確定
不採用選考の結果お断り
辞退候補者側から辞退
見送り条件不一致等で見送り
保留判断保留中

権限管理（管理者のみ）

  • 管理者: 全操作可能＋ユーザー追加/削除/権限変更
  • 編集者: 候補者のステータス・エリア・メモ等を編集可能
  • 閲覧者: 閲覧のみ（編集ボタン非表示）
  • 初回ログインユーザーが自動的に管理者になります
  • 未登録ユーザーはログインできません（管理者が事前に追加する必要あり）

技術仕様

  • Firebase PJ: wise-recruit-db5f3（個人情報分離のため独立プロジェクト）
  • 認証: Firebase Auth（Google OAuth）
  • DB: Cloud Firestore（asia-northeast1）
  • ホスティング: GitHub Pages（wiseinc-pharmacy/recruit-dashboard）
  • リアルタイム更新: Firestore onSnapshot（ページを開いている間は自動反映）
  • yui連携: recruit_bot.mjsがfirebase-admin SDKでFirestoreに書き込み

注意事項

  • yui自動更新は「新規メール受信時」と「Slackボタン操作時」のみ。スレッド内のやり取りを読んで判断する更新は手動です
  • 候補者の個人情報を含むため、権限のないユーザーを追加しないでください
  • 備考メモは誰が書いたか記録されません。重要な判断は日付と名前を添えて記入してください
